Test av format: Tillgänglighetsredogörelse

Tillgänglighetsredogörelse

Brister i m.fl.


Datum: 2020-11-05

Förteckning av hinder

  • 4.3 Hinder relaterade till visuell funktionsvariation (partiell och fullständig).
  • 4.3.1 Varseblivning om påbörjad laddning.
  • 4.3.2 Varseblivning om utdragen laddning.
  • 4.3.3 Varseblivning med popuper.
  • 4.3.4 Comboxar är inte passivt stängbara - när man stegar igenom lämnar man oavsiktliga spår.
  • 4.3.5 Navigation i huvudmenyn - går inte att tabba sig fram konsekvent, det är otydligt vad som är huvud- och undermenyer.
  • 4.3.6 Uppslag av kund via personnummer - dirty state-problem, labels på fält är dubbla.
  • 4.3.7 Uppslag av kund via personnummer utifrån myndighetsbehov - saknar information om fältets innehåll.
  • 4.3.8 Uppslag av kund via kundnummer - tabbningfel.
  • 4.3.9 Uppslag av kund via epost - alt-g + 2 = @-tecken-problem. Problem med Kortkommando-funktionen i Odoo som använder "ALT"-tangenten.
  • 4.3.10 Navigera mellan flikar på arbetsytan - där flikarna mina möten och kollegors möten visas som en tabell istället för två flikar.
  • 4.3.11 Läsa fält på fliken kontakter på kundbild visar bara en bild, inte text till den egna angivna adressen.
  • 4.3.12 Läsa uppgifter på fliken Kontakter och adresser visar bara en bild, inte text till den egna angivna adressen. Pop-up:er syns inte.
  • 4.3.13 Läsa uppgifter på fliken Snabblänkar visa som kanbankort och är därmed inte läsbara.
  • 4.3.14 Läsa uppgifter på fliken Programbeslut i AIS-å - det går inte att förstå att tabellen ska vara tom när den är tom.
  • 4.3.15 Läsa uppgifter på fliken Kundens planering - otydligt att man ska klicka på fliken och sedan på knappen, otydligt laddar, och laddat färdigt.
  • 4.3.16 Läsa information om kundens lokalkontor - otydlig ettikett och mycket otydlig vy för kontorsinformatione.
  • 4.3.17 Läsa information om kundens handläggare - sidan visas otydligt eftersom den skrivs som en tabell.
